## Phase 2: 욕망 발동 (Eros Activation)

> "에로스는 아름다운 것 안에서 낳고자 하는 욕망이다.
> 몸으로 낳는 것이 아니라 영혼으로 낳는 것이다." — 향연 206b-e

Phase 1에서 인식한 결핍을 **구체적 기능 욕망**으로 변환한다.
결핍이 욕망으로 변하는 순간, 에로스가 점화된다.

```
소크라테스 원본             →   포이에테스 변환
──────────────────────────────────────────────
Q4: "핵심 기능 3가지?"       →   Q6: "결핍 해소를 위한 핵심 기능 3가지?"

Q6: "{기능2} 왜?"           →   Q8: "왜 {기능2}를 욕망하는가?"
Q7: "{기능3} 왜?"           →   Q9: "왜 {기능3}을 욕망하는가?"
Q8: "이 기능 없이 성립?"     →   Q10: "이 기능들로 결핍이 해소되는가?"
```

## 5개 욕망 질문 상세

### Q6: 욕망의 구체화

**핵심 질문**: "결핍을 해소하기 위한 핵심 기능 3가지는?"

```json
{
  "questions": [{
    "question": "결핍을 인식했습니다. 이제 그 결핍을 해소할 구체적 기능을 찾아봅시다.\n\n{프로젝트명}에서 가장 핵심적인 기능 3가지가 뭐라고 생각하세요?",
    "header": "핵심 기능",
    "options": [
      {"label": "잘 모르겠어요", "description": "힌트를 주세요"},
      {"label": "1가지만 있어요", "description": "핵심 기능이 하나뿐이에요"},
      {"label": "3가지 이상", "description": "직접 입력하겠습니다 (Other)"}
    ],
    "multiSelect": false
  }]
}
```

**레벨별 접근:**

- **L1 (초심자)**: 구체적 예시를 제공
  ```
  "가계부 앱이라면... 보통 이런 기능들이 핵심이에요:
   1. 지출 기록 (돈이 나간 걸 적는 것)
   2. 지출 분석 (어디에 돈을 많이 썼는지 보는 것)
   3. 예산 알림 (돈을 너무 많이 쓰면 알려주는 것)

   이 중에서 가장 중요한 건 뭔가요? 아니면 다른 기능이 있나요?"
  ```

- **L2 (일반인)**: 카테고리 힌트 제공
  ```
  "핵심 기능을 찾을 때 이렇게 생각해보세요:
   1. 사용자가 가장 먼저 하는 행동 = 기능 1
   2. 그 다음에 보고 싶어하는 것 = 기능 2
   3. 없으면 절대 안 되는 것 = 기능 3"
  ```

- **L3/L4 (경험자/전문가)**: 자유 답변
  ```
  "핵심 기능 3가지를 알려주세요."
  ```

### Q7-Q9: 욕망의 근원 (Why 질문)

각 기능에 대해 "왜 이 기능을 욕망하는가?"를 물어 동기를 파악한다.

**패턴:**
```markdown
기능: "{기능명}"

Q: "왜 {기능명}을 욕망하시나요? 이것이 없으면 어떻게 되나요?"

사용자 답변 분석:
- 구체적 답변 → 다음 기능으로 진행
- 모호한 답변 → Why 2차 질문 (최대 2회)

예시:
기능: "지출 기록"
→ "왜 지출을 기록하고 싶어요?" (Why 1)
→ 사용자: "돈이 어디로 새는지 모르겠어서요"
→ "그걸 알면 뭘 하실 건가요?" (Why 2 - 행동 유도)
→ 사용자: "불필요한 지출을 줄이려고요"
→ [완료] 기능의 목적 확인 → 다음 기능으로
```

**Why 질문 연쇄 규칙:**
```
1회 Why: 기본 (모든 레벨)
2회 Why: 답변이 모호할 때 (L1/L2)
3회 Why: 절대 금지 (사용자 피로)
```

**에로스 언어로 변환:**
```
소크라테스: "왜 이 기능을 원하세요?"
포이에테스: "왜 이 기능을 욕망하시나요? 이것이 없으면 결핍이 해소되지 않나요?"
```

### Q10: 욕망의 완전성 검증

**핵심 질문**: "이 기능들이 모두 갖춰지면, 결핍이 해소되나요?"

```json
{
  "questions": [{
    "question": "지금까지 나온 핵심 기능을 정리해볼게요:\n\n1. {기능1}: {목적}\n2. {기능2}: {목적}\n3. {기능3}: {목적}\n\n이 기능들이 모두 있으면, 처음에 말한 결핍이 해소되나요?",
    "header": "욕망 검증",
    "options": [
      {"label": "네, 충분해요", "description": "이 기능들이면 결핍이 해소됩니다"},
      {"label": "부족해요", "description": "추가 기능이 필요합니다"},
      {"label": "과해요", "description": "기능을 줄여도 될 것 같아요"}
    ],
    "multiSelect": false
  }]
}
```

## 미니 디오티마 사다리 (욕망의 계층)

Phase 2에서 욕망을 단순히 나열하지 않고, 디오티마 사다리의 구조로 계층화한다.

```
Level 5: 의도/감정   — "왜 이것을 만들고 싶은가?" (이데아)
    ↑
Level 4: 문제 정의   — "무슨 문제를 풀려는 건가?" (메타 학습)
    ↑
Level 3: 아키텍처    — "어떤 구조로 풀 것인가?" (도메인 전이)
    ↑
Level 2: 기능        — "구체적으로 무엇이 필요한가?" (일반화)
    ↑
Level 1: 실행        — "어떻게 만들 것인가?" (패턴)
```

### 사다리 적용 방법

대부분의 사용자는 **Level 2 (기능)** 에서 욕망을 표현한다:
- "게시물 작성 기능이 필요해요"
- "결제 기능이 있어야 해요"

포이에테스는 이를 **Level 4-5** 까지 끌어올린다:

```
사용자: "게시물 작성 기능이 필요해요" (Level 2)
    ↓
포이에테스: "게시물로 무엇을 공유하고 싶은 건가요?" (Level 3으로 상승)
    ↓
사용자: "전문 지식을 공유하고 싶어요"
    ↓
포이에테스: "전문 지식 공유가 왜 중요한가요?" (Level 4로 상승)
    ↓
사용자: "제가 아는 걸 나누면 커뮤니티가 성장하니까요"
    ↓
포이에테스: "커뮤니티 성장이 당신에게 어떤 의미인가요?" (Level 5로 상승)
    ↓
사용자: "함께 배우고 성장하는 공간을 만들고 싶어요"
    ↓
[이데아 포착] → "함께 배우고 성장하는 공간" = 프로젝트의 핵심 가치
```

**L1/L2는 Level 3까지만, L3/L4는 Level 5까지 상승.**

## 답변 분석 방법

### 좋은 답변 패턴

```
✅ 구체적 답변:
- "사용자가 지출을 기록하고, 카테고리별로 분석하고, 예산 초과 시 알림받는 것"
- "실시간 협업, 버전 관리, 댓글 달기"

→ 다음 질문으로 진행
```

### 나쁜 답변 패턴

```
❌ 모호한 답변:
- "편리한 서비스를 만들고 싶어요"
- "사용자가 필요한 걸 할 수 있어요"

→ "어떤 부분이 편리해야 하나요? 구체적인 상황을 떠올려보세요."
→ Why 질문으로 구체화 유도
```

### 동적 질문 생성 규칙

```
규칙 1: 답변 기반 질문
사용자 답변: "SNS를 만들고 싶어요"
→ "SNS에서 사용자가 가장 먼저 하는 행동이 뭔가요?"

규칙 2: 모호한 답변 구체화
사용자 답변: "편리한 서비스"
→ "어떤 부분이 편리해야 하나요? 구체적인 예시를 들어주세요"

규칙 3: 기능 확장 질문
사용자 답변: "로그인 기능"
→ "소셜 로그인도 필요한가요? (Google/GitHub/Email)"
```

## Phase 2 완료 기준

```
✅ 핵심 기능 3가지가 구체적으로 도출됨
✅ 각 기능의 Why (욕망의 근원)가 파악됨
✅ 기능들이 결핍을 해소하는지 검증됨
✅ (L3/L4) 욕망의 계층이 Level 4-5까지 상승됨
```

→ 모두 충족 시 Phase 3 (다이몬 중재)로 진행

### 컨텍스트 캐싱

```
Phase 2 완료 시 기록:
- 핵심 기능 3가지 + 각 기능의 목적
- 각 기능의 Why 답변
- (옵션) 이데아 = 프로젝트의 핵심 가치
- 욕망 완전성 검증 결과
```

```
Q10: "지출 기록, 카테고리 분석, 예산 알림 — 이 3가지면 결핍이 해소되나요?"
A10: "네, 충분해요!"

→ Phase 3으로 진행
```
