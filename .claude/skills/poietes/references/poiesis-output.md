## Phase 4: 출산과 불멸 (Poiesis & Athanasia)

> "인간이 불사를 획득하는 유일한 방법은 낳는 것이다.
> 몸으로 낳는 것은 자식이고, 영혼으로 낳는 것은 지혜와 덕이다.
> 호메로스와 헤시오도스의 시가 불사인 것은
> 그것이 세대를 넘어 살아남기 때문이다." — 향연 208e-209e

Phase 1-3의 모든 대화 결과를 7개 기획 문서로 출산(Poiesis)하고,
출산물이 세대를 넘어 살아남도록 불멸 검증(Athanasia)한다.

## 불멸 검증 (Athanasia): 망하는 기획 4요건

> "호메로스의 시가 불사인 것은 그것이 세대를 넘어 살아남기 때문이다."

기획 문서가 "불사"하려면, 4가지 요건을 충족해야 한다.

### 검증 프로세스

```
┌─────────────────────────────────────────────────────────────────┐
│                  불멸 검증 (Athanasia Check)                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1️⃣ 벤치마킹 검사                                               │
│     └── PRD에 유사 서비스 3개 분석이 있는가?                     │
│     └── 차별화 포인트가 "다르다"가 아닌 "낫다"로 설명되는가?     │
│                                                                  │
│  2️⃣ 가설/사실 검사                                              │
│     └── PRD에 "가설 vs 사실" 테이블이 있는가?                    │
│     └── 각 가설에 검증 계획이 있는가?                            │
│                                                                  │
│  3️⃣ 타협 검사                                                   │
│     └── 핵심 가치(이데아)가 명확히 정의되어 있는가?              │
│     └── 타협한 부분이 기록되어 있는가?                           │
│                                                                  │
│  4️⃣ 실험 검사                                                   │
│     └── MVP가 구체적으로 정의되어 있는가?                        │
│     └── 실패 시나리오와 학습 목표가 있는가?                      │
│                                                                  │
│  ❌ 하나라도 미충족 시 → 보완 질문                                │
│  ✅ 모든 검사 통과 시 → 다음 단계로                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### AskUserQuestion 검증 질문

```json
{
  "questions": [{
    "question": "기획 문서를 검토했습니다.\n\n다음 중 보완이 필요한 부분이 있다면 선택해주세요:\n\n- 벤치마킹: 유사 서비스 분석이 충분한가요?\n- 검증: 가설과 사실이 구분되어 있나요?\n- 타협: 핵심 가치(이데아)가 유지되고 있나요?\n- 실험: MVP와 실패 대응이 준비되어 있나요?",
    "header": "불멸 검증",
    "options": [
      {"label": "모두 충분해요", "description": "검증 완료, 출산물이 불멸할 준비가 되었습니다"},
      {"label": "벤치마킹 부족", "description": "유사 서비스 분석 추가"},
      {"label": "가설 검증 부족", "description": "가설/사실 구분 추가"},
      {"label": "MVP 불명확", "description": "최소 출산물 재정의 필요"}
    ],
    "multiSelect": true
  }]
}
```

### 보완이 필요한 경우

선택된 부족 항목에 대해 추가 질문을 진행한다:

```
"벤치마킹 부족" 선택 시:
→ "비슷한 서비스를 3개 알려주세요. 그 서비스와 당신의 차별점은 뭔가요?"

"가설 검증 부족" 선택 시:
→ "기획에서 '이럴 것이다'라고 가정한 부분이 있나요? 그 가정을 어떻게 검증할 수 있나요?"

"MVP 불명확" 선택 시:
→ "딱 1가지 기능만 출시한다면 어떤 기능인가요? 그 기능만으로 사용자가 만족할까요?"
```
