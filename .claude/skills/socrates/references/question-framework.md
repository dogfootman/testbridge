# 21개 질문 체계 (Question Framework)

> **로드 시점**: 소크라테스 대화 진행 시 (Phase 1)

---

## 카테고리별 질문 구조

### 1. 핵심 가치 (3개)
1. "무엇을 만들고 싶어요?"
2. "이 서비스에서 사용자가 가장 먼저 하는 행동이 뭔가요?"
3. "이 서비스 없이는 절대 안 되는 기능이 뭔가요?"

### 2. 기능 탐색 (5개)
4. "핵심 기능 3가지가 뭐예요?"
5. "{기능1}을 왜 하고 싶어요?" (Why 질문)
6. "{기능2}를 왜 하고 싶어요?" (Why 질문)
7. "{기능3}을 왜 하고 싶어요?" (Why 질문)
8. "이 기능들이 없으면 서비스가 성립 안 되나요?"

### 3. 화면 매핑 (5개)
9. "{기능1}은 어떤 화면에서 하나요?"
10. "{기능2}는 어떤 화면에서 하나요?"
11. "{기능3}은 어떤 화면에서 하나요?"
12. "화면 간 이동 경로가 어떻게 되나요?"
13. "각 화면에 어떤 버튼/입력 필드가 필요한가요?"

### 4. 사용자 흐름 (3개)
14. "사용자가 처음 들어왔을 때 뭘 보나요?"
15. "그 다음엔 뭘 하나요?"
16. "마지막으로 뭘 하고 떠나나요?"

### 5. 기술 스택 (3개)
17. "어떤 기술 스택을 쓰고 싶어요?" (L3/L4만)
18. "왜 그 기술을 선택했나요?" (L3/L4만)
19. "다른 기술과 비교해 보셨나요?" (L3/L4만)

### 6. 제약 조건 (2개)
20. "시간/예산/인력 제약이 있나요?"
21. "MVP로 먼저 만들 기능은 뭔가요?"

## 레벨별 질문 수 조정

| 레벨 | 질문 수 | 생략 질문 |
|------|---------|----------|
| L1 (초심자) | 18~21개 | 기술 스택 질문 (17-19) |
| L2 (일반인) | 15~18개 | 기술 스택 질문 일부 |
| L3 (경험자) | 12~15개 | 중복 Why 질문 |
| L4 (전문가) | 10~12개 | 기초 질문, 중복 Why 질문 |

## 답변 분석 방법

### 좋은 답변 패턴
```
✅ 구체적 답변:
- "사용자가 로그인 → 대시보드 → 프로젝트 생성"
- "핵심 기능은 실시간 협업, 버전 관리, 알림"

→ 다음 질문으로 진행
```

### 나쁜 답변 패턴
```
❌ 모호한 답변:
- "편리한 서비스를 만들고 싶어요"
- "사용자가 필요한 걸 할 수 있어요"

→ Why 질문으로 구체화 유도
```

### Why 질문 연쇄 규칙
```
1회 Why: 기본 (모든 레벨)
2회 Why: 답변이 모호할 때 (L1/L2)
3회 Why: 절대 금지 (사용자 피로)
```

## 동적 질문 생성 규칙

### 규칙 1: 답변 기반 질문
```
사용자 답변: "SNS를 만들고 싶어요"
→ 동적 질문: "SNS에서 가장 중요한 기능이 뭔가요? (게시물/친구/메시지/피드)"
```

### 규칙 2: 모호한 답변 구체화
```
사용자 답변: "편리한 서비스"
→ 동적 질문: "어떤 부분이 편리해야 하나요? 구체적인 예시를 들어주세요"
```

### 규칙 3: 기능 확장 질문
```
사용자 답변: "로그인 기능"
→ 동적 질문: "소셜 로그인도 필요한가요? (Google/GitHub/Email)"
```

## 질문 흐름 예시

### 시나리오: SNS 기획

```
Q1: "무엇을 만들고 싶어요?"
A1: "SNS를 만들고 싶어요"

Q2: "핵심 기능 3가지가 뭐예요?"
A2: "게시물 작성, 친구 추가, 좋아요"

Q3: "게시물 작성을 왜 하고 싶어요?"
A3: "사용자가 생각을 공유하기 위해"

Q4: "게시물 작성은 어떤 화면에서 하나요?"
A4: "메인 피드 화면에 작성 버튼이 있어요"

Q5: "작성 버튼을 누르면 어떤 화면으로 이동하나요?"
A5: "작성 모달이 뜨고, 텍스트/이미지 입력 후 게시"

Q6: "화면 간 이동 경로가 어떻게 되나요?"
A6: "로그인 → 피드 → 작성 모달 → 피드 갱신"

Q7: "각 화면에 어떤 컴포넌트가 필요한가요?"
A7: "피드: 게시물 카드, 작성 버튼 / 모달: 텍스트 에디터, 이미지 업로드"
```

## 질문 회피 방지 기법

### 패턴 1: 기계적 질문 금지
```
❌ 나쁜 예:
Q1: "무엇을 만들고 싶어요?"
Q2: "이 서비스에서 사용자가 가장 먼저 하는 행동이 뭔가요?"
Q3: "핵심 기능 3가지가 뭐예요?" (기계적 순서)

✅ 좋은 예:
Q1: "무엇을 만들고 싶어요?"
A1: "SNS를 만들고 싶어요"
Q2: "SNS에서 가장 중요한 기능이 뭔가요?" (답변 기반 동적 질문)
```

### 패턴 2: 답변 요약 후 질문
```
✅ 좋은 예:
"게시물 작성, 친구 추가, 좋아요 기능을 말씀하셨네요.
그럼 이 기능들이 어떤 화면에서 동작하는지 상상해볼까요?"
```

## 질문 종료 조건

### 완료 기준
```
✅ 모든 Phase 완료:
- Phase 0: 레벨 측정 완료
- Phase 1: 핵심 기능 3가지 도출
- Phase 2: 모든 기능이 화면에 매핑됨
- Phase 2.5: 기술 스택 결정
- Phase 3: 7개 문서 생성 완료
- Phase 3.5: 기획 검증 완료
```

### 조기 종료 방지
```
❌ 금지:
- "기능이 너무 간단해서 질문 생략" → 무조건 Phase 2 진행
- "사용자가 화면을 모르면 스킵" → 화면 예시 제공 후 재질문
- "기술 스택을 정하지 못하면 보류" → L1/L2는 자동 추천
```
