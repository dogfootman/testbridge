# 디오티마 사다리 (Phase 4) — 핵심 레퍼런스

> **로드 시점**: Phase 4 진입 시 (가장 핵심적인 레퍼런스)

---

## 디오티마의 가르침

> "이것이 사랑의 신비에 입문하는 올바른 방법이다.
> 하나의 아름다운 몸에서 시작하여, 둘로, 모든 아름다운 몸으로,
> 그 다음 아름다운 영혼으로, 그 다음 아름다운 활동과 법으로,
> 그 다음 아름다운 지식으로, 마침내 아름다움 그 자체에 이르는 것이다."
> — 향연 211c

## 6계층 추상화 사다리

```
Layer 5: 불변 원리 = 이데아 (Invariant Form)          ← "아름다움 그 자체"
    ↑
Layer 4: 학습 방법 자체의 학습 (Meta-Learning)         ← "아름다운 지식"
    ↑
Layer 3: 다른 도메인으로 전이 (Transfer)               ← "아름다운 법"
    ↑
Layer 2: 모든 사례를 포괄하는 규칙 (Generalization)    ← "아름다운 활동"
    ↑
Layer 1: 여러 사례의 패턴 (Feature Extraction)         ← "모든 아름다운 몸"
    ↑
Layer 0: 하나의 구체적 사례 (Raw Data)                 ← "하나의 아름다운 몸"
```

---

## Layer별 상세

### Layer 0: 하나의 구체적 사례 (Raw Data)

**목표**: 문제를 하나의 구체적 사례로 환원한다.

**행동**:
1. Phase 3 탐색 결과에서 가장 구체적인 사례 1개를 선택
2. 사례를 완전히 이해할 때까지 분해:
   - 코드라면: 해당 함수/모듈을 Read로 완전히 읽기
   - 개념이라면: 구체적 입출력 예시 확보
   - 버그라면: 정확한 재현 경로 확보
3. "이 사례를 완벽히 설명할 수 있는가?" 자문

**AskUserQuestion**:
```json
{
  "questions": [{
    "question": "Layer 0 구체적 사례를 확인합니다. 이 사례가 문제를 잘 대표하나요?",
    "header": "Layer 0",
    "options": [
      {"label": "적절하다", "description": "이 사례가 문제의 핵심을 담고 있다"},
      {"label": "다른 사례 추가", "description": "이것만으로는 부족하다, 더 필요하다"},
      {"label": "사례 변경", "description": "이 사례가 적절하지 않다, 다른 것으로"}
    ],
    "multiSelect": false
  }]
}
```

**상승 기준**: 사례를 완벽히 설명할 수 있으면 Layer 1로 상승

---

### Layer 1: 여러 사례의 패턴 (Feature Extraction)

**목표**: 여러 구체적 사례에서 공통 패턴을 추출한다.

**행동**:
1. Layer 0 사례와 유사한 사례를 2-3개 더 찾는다
   - Grep으로 유사 패턴 검색
   - 같은 유형의 에러/동작 검색
2. 사례들의 공통점/차이점 정리
3. "이 패턴이 모든 사례를 설명하는가?" 자문

**패턴 추출 템플릿**:
```markdown
### 관찰된 사례
| # | 사례 | 공통점 | 차이점 |
|---|------|--------|--------|
| 1 | {Layer 0 사례} | {공통} | {차이} |
| 2 | {추가 사례} | {공통} | {차이} |
| 3 | {추가 사례} | {공통} | {차이} |

### 추출된 패턴
- 패턴 A: {공통점에서 추출한 패턴}
- 패턴 B: {또 다른 패턴}
```

**AskUserQuestion**: Layer 1→2 상승 여부 확인
```json
{
  "questions": [{
    "question": "Layer 1 패턴을 확인합니다. 이 패턴이 관찰된 사례를 충분히 설명하나요?",
    "header": "Layer 1",
    "options": [
      {"label": "패턴이 충분하다", "description": "모든 사례를 설명할 수 있다 → Layer 2로"},
      {"label": "사례 더 필요", "description": "아직 예외 사례가 있다 → Layer 1 유지"},
      {"label": "패턴 수정 필요", "description": "패턴이 잘못 추출되었다 → Layer 1 재작업"}
    ],
    "multiSelect": false
  }]
}
```

**상승 기준**: 패턴이 관찰된 모든 사례를 설명하면 Layer 2로 상승

---

### Layer 2: 모든 사례를 포괄하는 규칙 (Generalization)

**목표**: 패턴을 "만약 X이면 항상 Y" 형태의 규칙으로 일반화한다.

**행동**:
1. Layer 1 패턴을 규칙으로 변환
2. 반례 검색: 이 규칙이 깨지는 경우를 적극 탐색
   - `/the-fool` Mode 1 (가정 노출) 원칙 적용: "이 규칙의 전제 조건은 무엇인가?"
3. 규칙의 적용 범위(scope)와 한계(limitation) 명시

**규칙 정의 템플릿**:
```markdown
### 일반화된 규칙

**규칙**: 만약 {조건}이면, {결과}이다.
**적용 범위**: {이 규칙이 유효한 컨텍스트}
**한계**: {이 규칙이 깨지는 조건}
**반례**: {찾은 반례와 규칙 수정 여부}
```

**AskUserQuestion**: Layer 2→3 상승 여부 확인

**상승 기준**: 규칙이 알려진 모든 사례에 적용되고, 반례에 대한 설명이 가능하면 Layer 3로 상승

---

### Layer 3: 다른 도메인으로 전이 (Transfer)

**목표**: Layer 2의 규칙이 완전히 다른 도메인에서도 성립하는지 검증한다.

**행동**:
1. 현재 도메인의 규칙을 추상화하여 표현
2. 유사 구조를 가진 다른 도메인 탐색:
   - 소프트웨어 → 건축, 생물학, 경제학 등의 유사 패턴
   - 프론트엔드 → 백엔드, 인프라 등의 유사 문제
3. 전이 성공/실패 기록

**도메인 전이 템플릿**:
```markdown
### 도메인 전이 검증

**원래 도메인**: {현재 문제 도메인}
**규칙**: {Layer 2에서 도출한 규칙}

| 전이 대상 도메인 | 유사 구조 | 규칙 적용 가능? | 근거 |
|-----------------|----------|---------------|------|
| {도메인 A} | {유사점} | 예/아니오 | {설명} |
| {도메인 B} | {유사점} | 예/아니오 | {설명} |

**전이 결과**: {규칙의 보편성 평가}
```

**AskUserQuestion**: Layer 3→4 상승 여부 확인

**상승 기준**: 최소 1개 다른 도메인에서 규칙이 성립하면 Layer 4로 상승. 전이 실패 시 → Layer 2로 내려가 규칙 수정

---

### Layer 4: 학습 방법 자체의 학습 (Meta-Learning)

**목표**: "어떻게 이 규칙을 발견했는가?"를 분석한다. 규칙이 아니라 규칙을 발견하는 방법을 학습한다.

**행동**:
1. Layer 0→3의 상승 과정 자체를 분석
2. "어떤 질문이 핵심 돌파구였는가?"
3. "어떤 관점 전환이 패턴 발견을 가능하게 했는가?"
4. 메타 패턴 추출: "비슷한 문제를 만나면 어떻게 접근해야 하는가?"

**메타 학습 템플릿**:
```markdown
### 메타 학습

**핵심 돌파구**: {어떤 질문/관점이 패턴 발견을 촉발했는가?}
**상승 경로 분석**:
- Layer 0→1: {어떤 행동이 패턴 추출을 가능하게 했는가?}
- Layer 1→2: {어떤 질문이 일반화를 촉발했는가?}
- Layer 2→3: {어떤 관점 전환이 도메인 전이를 가능하게 했는가?}

**메타 패턴**: 비슷한 유형의 문제를 만나면 → {접근법}
**재사용 가능한 질문**: {문제 해결에 반복 사용 가능한 질문 목록}
```

**AskUserQuestion**: Layer 4→5 상승 여부 확인

**상승 기준**: 메타 패턴이 다른 문제에도 적용 가능한 형태로 추출되면 Layer 5로 상승

---

### Layer 5: 불변 원리 = 이데아 (Invariant Form)

**목표**: 모든 계층을 관통하는 불변의 원리를 포착한다. 이것은 구체적 사례에 의존하지 않는 순수한 형식이다.

**행동**:
1. Layer 0~4의 모든 결과를 조망
2. "이 모든 것을 하나의 문장으로 표현하면?"
3. 불변 원리의 3가지 조건 검증:
   - **보편성**: 관찰된 모든 사례에 적용되는가?
   - **필연성**: 이 원리 없이 사례를 설명할 수 없는가?
   - **불변성**: 맥락이 바뀌어도 원리가 유지되는가?

**이데아 정의 템플릿**:
```markdown
### 이데아 (불변 원리)

**한 문장 정의**: {모든 계층을 관통하는 불변의 원리}

**3가지 조건 검증**:
- 보편성: {모든 사례에 적용되는가?} → {예/아니오 + 근거}
- 필연성: {이 원리 없이 설명 가능한가?} → {예/아니오 + 근거}
- 불변성: {맥락 변경 시 유지되는가?} → {예/아니오 + 근거}

**이 원리로부터 도출되는 것들**:
- Layer 0에서: {구체적 사례에 어떻게 나타나는가}
- Layer 2에서: {규칙으로 어떻게 표현되는가}
- Layer 3에서: {다른 도메인에서 어떻게 나타나는가}
```

## 사다리 운행 규칙

### 절대 규칙

1. **순차 상승만 허용**: Layer 0 → 1 → 2 → 3 → 4 → 5. 건너뛰기 절대 금지
2. **상승 기준 충족 필수**: 각 Layer의 상승 기준을 충족해야 다음 Layer로
3. **하강은 자유**: Layer 3에서 반례 발견 시 Layer 2로 하강 가능
4. **AskUserQuestion 필수**: 매 Layer 상승 전 사용자 확인

### Small Cycle (추상 재상승)

Phase 5 (창작) 중 산출물이 Layer 5 원리를 제대로 반영하지 못하면:
1. 어느 Layer에서 괴리가 발생했는지 진단
2. 해당 Layer로 돌아가 재작업
3. 다시 상승하여 Phase 5로 복귀

```
Phase 5 → "원리 미반영" 감지 → Layer 3 복귀 → Layer 3→4→5 재상승 → Phase 5 복귀
```

### `/the-fool` 연동

각 Layer에서 `/the-fool` Mode 1 (가정 노출) 원칙을 적용:
- **Layer 1**: "이 패턴이 정말 공통점인가, 우연의 일치인가?"
- **Layer 2**: "이 규칙의 숨은 전제 조건은 무엇인가?"
- **Layer 3**: "도메인 전이가 가능하다고 판단한 근거는 충분한가?"
- **Layer 4**: "이 메타 패턴이 편향(confirmation bias)은 아닌가?"
- **Layer 5**: "이것이 진짜 불변인가, 내 관점의 한계인가?"

### `--ladder-only` 모드

Phase 1-3을 건너뛰고 Phase 4만 실행:
- 이미 구체적 사례(Layer 0 데이터)가 있을 때 사용
- AskUserQuestion으로 Layer 0 사례를 직접 입력받음
- 나머지 Layer 1-5는 동일하게 순차 상승
