# 창작 Poiesis (Phase 5)

> **로드 시점**: Phase 5 진입 시 (디오티마 사다리 완료 후)

---

## 헤파이스토스의 장인정신

> "에로스는 아름다운 것 안에서 낳고자 하는 욕망이다.
> 몸으로 낳는 것이 아니라 영혼으로 낳는 것이다." — 향연 206b-e

Phase 4에서 도달한 추상 원리(이데아)를 구체적 산출물로 변환한다.
헤파이스토스(신들의 대장장이)가 이데아의 불꽃을 형태로 빚듯이, 추상을 구체로 단조한다.

## 산출물 유형

Phase 4의 결과에 따라 적절한 산출물 유형을 선택한다:

| 산출물 유형 | 적용 조건 | 예시 |
|-----------|----------|------|
| 코드 수정 | Layer 5 원리가 코드 변경을 요구할 때 | 리팩토링, 버그 수정, 신규 기능 |
| ADR (Architecture Decision Record) | 아키텍처 결정이 필요할 때 | 기술 선택, 패턴 도입 |
| 분석 문서 | 이해/통찰이 주요 산출물일 때 | 성능 분석, 원인 분석 |
| 설계 패턴 | Layer 3-5에서 재사용 가능한 패턴 도출 시 | 디자인 패턴, 코드 패턴 |
| 워크플로우 | 프로세스 개선이 필요할 때 | 배포 파이프라인, 리뷰 프로세스 |

## 창작 절차

### Step 1: 원리 → 산출물 매핑

```markdown
### 이데아 → 산출물 매핑

**Layer 5 원리**: {한 문장}
**산출물 유형**: {코드/ADR/분석/패턴/워크플로우}
**구체화 계획**:
1. {원리의 어떤 측면을 반영하는가}
2. {구체적으로 무엇을 만드는가}
3. {완성 기준은 무엇인가}
```

### Step 2: 구현

산출물 유형에 따라 실행:

**코드 수정 시**:
1. 변경 대상 파일 Read
2. Layer 5 원리에 따라 Edit/Write
3. 변경 범위를 최소화 (over-engineering 금지)

**ADR 작성 시**:
```markdown
# ADR-{번호}: {제목}

## 상태: 제안됨

## 맥락
{Phase 1-3에서 발견한 결핍과 탐색 결과}

## 결정
{Layer 5 원리에 기반한 결정}

## 근거
- Layer 2 규칙: {일반화된 규칙}
- Layer 3 전이: {다른 도메인에서의 검증}
- Layer 5 원리: {불변 원리}

## 결과
{예상되는 긍정적/부정적 결과}
```

**분석 문서 시**: → Phase 6의 eros-analysis.md 템플릿 활용

### Step 3: Verification Discipline

> **"아마 작동할 것" 금지. 증거 없이 완료를 주장하지 않는다.**

산출물 유형별 검증:

| 산출물 유형 | 검증 방법 |
|-----------|----------|
| 코드 수정 | 테스트 실행 (pytest/npm test 등), 빌드 확인, lint 통과 |
| ADR | Layer 2 규칙과의 일관성 확인, 반례 검토 |
| 분석 문서 | Grep/Read으로 주장의 증거 재확인 |
| 설계 패턴 | 최소 2개 사례에서 패턴 적용 가능성 확인 |
| 워크플로우 | 단계별 실행 가능성 확인 |

## Small Cycle 판단

산출물 완성 후 자문:

> "이 산출물이 Layer 5 원리를 제대로 반영하는가?"

**반영 확인 체크리스트**:
- [ ] Layer 5 원리의 보편성이 산출물에 반영되었는가?
- [ ] Layer 5 원리의 필연성이 산출물에 반영되었는가?
- [ ] Layer 5 원리의 불변성이 산출물에 반영되었는가?
- [ ] Layer 2 규칙과 모순되지 않는가?

**하나라도 "아니오"이면** → Small Cycle 발동:
1. 괴리가 발생한 Layer 식별
2. 해당 Layer로 돌아가 재작업
3. Layer 재상승 후 Phase 5 복귀

```
Phase 5 → 괴리 감지 (예: Layer 3 전이가 불완전)
    → Layer 3 복귀 → 추가 도메인 전이 검증
    → Layer 3→4→5 재상승
    → Phase 5 복귀 → 산출물 수정
```

## Phase 5 완료 기준

- [ ] 산출물이 Layer 5 원리를 반영한다는 증거 제시
- [ ] Verification Discipline 통과 (추측 표현 없음)
- [ ] Small Cycle 필요 여부 판단 완료
- [ ] 산출물이 Phase 2의 목표 상태에 부합

→ 모두 충족 시 Phase 6 (불사 보존)으로 진행
