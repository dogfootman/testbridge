# 통합 시나리오 테스트 매뉴얼

**프로젝트**: TestBridge - Google Play 테스터 매칭 플랫폼
**작성일**: 2026-03-01
**버전**: 1.0

본 문서는 TestBridge의 End-to-End 통합 시나리오에 대한 수동 테스트 매뉴얼입니다.

---

## 시나리오 1: 개발자 전체 플로우

### 목적
개발자가 앱을 등록하고 테스터를 모집하여 피드백을 받는 전체 과정을 검증합니다.

**검증 범위**:
- 회원가입 및 Google OAuth 인증
- 앱 등록 (4단계 위저드)
- 대시보드 및 앱 관리
- 지원자 승인 프로세스
- 상태 자동 전환 (RECRUITING → IN_TESTING)
- 피드백 확인 및 관리

### 소요 시간
약 **15-20분** (테스터 역할 수행 시간 제외)

### 사전 준비사항
- [ ] Google 계정 (개발자용)
- [ ] 테스트할 앱 정보
  - 앱 이름 (예: MyApp)
  - 카테고리 (게임, 생산성 등)
  - 앱 설명 (최소 50자)
- [ ] 앱 스크린샷 3장 이상 (URL 또는 로컬 파일)
- [ ] Backend 서버 실행 중 (http://localhost:8000)
- [ ] Frontend 서버 실행 중 (http://localhost:3000)
- [ ] PostgreSQL 데이터베이스 실행 중

### 환경 설정 확인

```bash
# Backend 서버 확인
curl http://localhost:8000/health
# 응답: {"status": "ok"}

# Frontend 서버 확인
curl http://localhost:3000
# 응답: HTML 페이지
```

### 단계별 절차

#### Step 1: 회원가입 및 로그인

**작업**:
1. 브라우저에서 `http://localhost:3000` 접속
2. 랜딩 페이지에서 **"개발자로 시작하기"** 버튼 클릭
3. Google OAuth 로그인 페이지로 리다이렉트 확인
4. Google 계정 선택 (개발자용 계정)
5. 권한 동의 확인 (이메일, 프로필 정보)
6. 로그인 완료 후 리다이렉트 확인

**예상 결과**:
- [x] `/auth/signin` 페이지 표시
- [x] Google OAuth 동의 화면 표시
- [x] 로그인 성공 후 `/developer/dashboard`로 리다이렉트
- [x] 우측 상단에 프로필 이미지 및 이름 표시
- [x] 역할(Role)이 `DEVELOPER`로 설정됨

**확인 방법**:
```javascript
// 브라우저 콘솔에서 확인
fetch('/api/auth/session')
  .then(r => r.json())
  .then(console.log)
// user.role === "DEVELOPER" 확인
```

### 트러블슈팅

#### 문제 1: Google 로그인 실패

**증상**:
- "Sign in with Google" 버튼 클릭 시 에러
- 또는 리다이렉트 후 로그인 실패

**원인**:
- NextAuth 설정 오류
- Google OAuth 클라이언트 ID/Secret 미설정
- Redirect URI 불일치

**해결 방법**:
```bash
# 1. .env.local 파일 확인
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=your-secret-key

# 2. Google Cloud Console에서 Redirect URI 확인
# 허용된 리다이렉트 URI에 추가:
http://localhost:3000/api/auth/callback/google

# 3. 서버 재시작
npm run dev
```

**증상**:
- "앱 등록" 버튼 클릭 시 500 에러
- 또는 "네트워크 오류" 메시지

**원인**:
- Backend 서버 미실행
- Database 연결 오류
- 인증 토큰 만료

**해결 방법**:
```bash
# 1. Backend 서버 상태 확인
curl http://localhost:8000/health

# 2. Database 연결 확인
docker ps | grep postgres
# 또는
psql -h localhost -U testbridge -d testbridge

# 3. 로그아웃 후 재로그인
# 브라우저에서 /api/auth/signout 접속
```

#### 문제 3: 지원자 승인 버튼이 보이지 않음

**증상**:
- "지원자" 탭에 목록은 있지만 "승인" 버튼 없음

**원인**:
- 권한 부족 (본인이 등록한 앱이 아님)
- 이미 승인된 지원자

**해결 방법**:
```bash
# 1. 앱 소유자 확인 (브라우저 콘솔)
fetch('/api/apps/1')
  .then(r => r.json())
  .then(d => console.log(d.developerId))
# 현재 로그인한 사용자 ID와 일치해야 함

# 2. 지원 상태 확인
fetch('/api/applications?appId=1')
  .then(r => r.json())
  .then(console.log)
# status가 "PENDING"인지 확인
```

#### 문제 4: 피드백이 표시되지 않음

**증상**:
- "피드백" 탭이 비어 있음

3. 리워드 섹션 확인
   - 총 적립 리워드
   - 출금 가능 금액
   - 리워드 내역
4. 피드백 제출로 인한 리워드 지급 확인

**예상 결과**:
- [x] 총 리워드: 10,000원 (앱 리워드 금액)
- [x] 리워드 내역에 "MyTestApp 피드백 리워드" 표시
- [x] 지급 일시 정확히 표시

**참고**:
리워드 자동 지급 로직이 구현되어 있지 않은 경우, 개발자가 수동으로 지급해야 할 수 있습니다.

#### 문제 1: "지원하기" 버튼이 비활성화됨

**증상**:
- 앱 상세 페이지에서 "지원하기" 버튼 클릭 불가

**원인**:
- 이미 지원한 앱
- 모집 인원 마감
- 앱 상태가 RECRUITING이 아님

**해결 방법**:
```bash
# 1. 앱 상태 확인
curl http://localhost:8000/api/apps/1
# status가 "RECRUITING"인지 확인

# 2. 지원 여부 확인
curl http://localhost:8000/api/applications?appId=1&testerId=현재사용자ID
# 이미 지원한 경우 빈 배열 또는 기존 지원서 반환

# 3. 모집 인원 확인
# recruited / maxTesters 비교
```

#### 문제 2: 피드백 작성 페이지 접근 불가 (403 에러)

**증상**:

**해결 방법**:
```bash
# 1. 참여 상태 확인
curl http://localhost:8000/api/participations?appId=1&testerId=현재사용자ID
# status가 "APPROVED"인지 확인

# 2. 개발자에게 승인 요청
# 시나리오 1의 Step 9 참조

# 3. 참여 상태 수동 업데이트 (임시, 개발 환경만)
curl -X PATCH http://localhost:8000/api/participations/1 \
  -H "Content-Type: application/json" \
  -d '{"status": "APPROVED"}'
```

#### 문제 5: 리워드가 지급되지 않음

**증상**:
- 피드백 제출 완료했지만 리워드 0원

**원인**:
- 리워드 자동 지급 로직 미구현
- Backend 트리거 실패

**해결 방법**:
```bash
# 1. 리워드 지급 로직 확인
# backend/app/routers/feedbacks.py의 create_feedback 함수 확인

# 2. 수동으로 리워드 지급 (임시)
curl -X POST http://localhost:8000/api/rewards \
  -H "Content-Type: application/json" \
  -d '{
    "userId": 테스터ID,
    "amount": 10000,
    "type": "EARNED",
    "description": "MyTestApp 피드백 리워드"
  }'

# 3. 리워드 내역 확인
curl http://localhost:8000/api/rewards?userId=테스터ID
```
