# TestBridge 요구사항 정의서

**프로젝트명**: TestBridge - Google Play 비공개 테스트 매칭 플랫폼  
**버전**: v1.0  
**작성일**: 2026-02-27  
**작성자**: TestBridge 기획팀  

---

## 1. 문서 개요

본 문서는 TestBridge 서비스의 기능 및 비기능 요구사항을 정의한다.  
업무분류 체계에 따라 순번을 부여하며, MVP 범위와 2차 개발 범위를 구분한다.

### 업무분류 코드 체계

| 코드 | 업무분류 | 설명 |
|------|----------|------|
| COM | 공통 | 인증, 프로필, 알림 등 공통 기능 |
| DEV | 개발자 | 앱 등록, 테스트 관리 등 개발자 기능 |
| TST | 테스터 | 앱 탐색, 테스트 참여, 피드백 등 테스터 기능 |
| RWD | 리워드 | 포인트, 출금, 기프티콘, 크레딧 등 리워드 기능 |
| PAY | 결제 | 구독, 건별 결제, 결제 수단 관리 |
| ADM | 관리자 | 대시보드, 사용자/앱/정산/CS 관리 |
| CS | 고객지원 | 문의, FAQ, 공지사항, 신고 |
| NFR | 비기능 | 성능, 보안, 인프라 등 비기능 요구사항 |

### 우선순위 정의

| 구분 | 설명 |
|------|------|
| P1 | MVP 필수 (1차 개발) |
| P2 | 2차 개발 |
| P3 | 향후 고려 |
