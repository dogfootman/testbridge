# TestBridge 화면 흐름도 (User Flow)

**버전**: v1.0  
**작성일**: 2026-02-28  
**목적**: 13개 화면 간의 전체 이동 경로를 역할별/조건별로 정의. 구현 시 라우팅 설계의 입력으로 활용.

---

## 1. 전체 화면 흐름 (Master Flow)

```
                          ┌─────────────────────────────────────────────┐
                          │              S-01 랜딩                       │
                          │  [Google 로그인] [카카오] [네이버]           │
                          └──────────┬──────────────────────────────────┘
                                     │
                          ┌──────────▼──────────┐
                          │  Google OAuth 팝업    │
                          └──────────┬──────────┘
                                     │
                     ┌───────────────┼───────────────┐
                     │ 신규          │ 기존           │ 실패
                     ▼               ▼               ▼
              ┌──────────┐   ┌──────────────┐   S-01 (에러)
              │ S-02      │   │ 역할 확인     │
              │ 회원가입   │   └──┬───────┬──┘
              │           │      │       │
              │ ①역할선택 │      │       │
              │ ②약관동의 │      ▼       ▼
              │ ③프로필   │   개발자    테스터
              └─────┬─────┘      │       │
                    │            ▼       ▼
              ┌─────┼──────┐  D-01    T-01
              │     │      │
              ▼     ▼      ▼
           개발자  테스터   둘 다
              │     │      │
              ▼     ▼      ▼
           D-01   T-01   D-01
```

## 2. 비로그인 사용자 흐름

```
[최초 방문]
  │
  ▼
S-01 랜딩
  │
  ├─[Google 로그인]──→ OAuth ──→ 신규: S-02 / 기존: D-01 또는 T-01
  ├─[카카오 로그인]──→ OAuth ──→ (동일 흐름)
  ├─[네이버 로그인]──→ OAuth ──→ (동일 흐름)
  │
  ├─[FAQ 보기]──→ CS-01 (비로그인 접근 가능, FAQ만)
  │
  └─[인증 필요 페이지 접근]──→ S-01 리디렉션 (returnUrl 보존)
```

### 가입 세부 흐름

```
S-02 회원가입
  │
  ├─① 역할 선택 ─────────────────────────┐
  │   [DEVELOPER] [TESTER] [BOTH]        │
  │   미선택 → "완료" 비활성화            │
  │                                       │
  ├─② 약관 동의 ─────────────────────────┤
  │   [전체동의] [이용약관*] [개인정보*]   │
  │   [마케팅(선택)]                      │
  │   필수 미동의 → "완료" 비활성화       │
  │                                       │
  ├─③ 프로필 설정 ────────────────────────┤
  │   닉네임* / 프로필 이미지 / 자기소개   │
  │   중복/금칙어 → 인라인 에러           │
  │                                       │
  └─[가입 완료]                           │
       │                                  │
       ├─DEVELOPER → D-01                 │
       ├─TESTER → T-01                    │
       └─BOTH → D-01                      │
```

## 3. 개발자 흐름

### 3-1. 메인 네비게이션

```
[상단 Nav]
  ├─ 로고 (→ D-01)
  ├─ 대시보드 (→ D-01)
  ├─ 내 앱 (→ D-03)
  ├─ 결제/구독 (→ D-05)
  ├─ [역할 토글: 테스터 모드] (→ T-01)
  ├─ 🔔 알림 (→ 알림 센터 드롭다운)
  └─ 프로필 (→ S-04)
```

### 3-2. 대시보드 → 앱 등록 흐름

```
D-01 대시보드
  │
  ├─[+ 새 앱 등록]──→ 한도 확인
  │                     │
  │              ┌──────┼──────┐
  │              │ 한도 OK     │ 한도 초과
  │              ▼             ▼
  │           D-02 Step1    모달: "업그레이드"
  │              │              │
  │              │           D-05 결제
  │              │
  │           D-02 앱 등록 (4단계 위자드)
  │              │
  │      ┌───────┼───────┬───────┐
  │      ▼       ▼       ▼       ▼
  │   Step 1   Step 2   Step 3   Step 4
  │   기본정보  테스트설정 리워드   피드백
  │      │       │       │       │
  │      ├─다음→ ├─다음→  ├─다음→ ├─등록
  │      ├─◀이전 ├─◀이전  ├─◀이전 │
  │      │       │       │       │
  │      │  [CREDIT_EXCHANGE]    │
  │      │  → Step3 건너뜀       │
  │      │                       │
  │      └───────────────────────┤
  │                              │
  │              ┌───────────────┤
  │              │ PAID_REWARD   │ CREDIT_EXCHANGE
  │              ▼               ▼
  │        결제 화면          크레딧 차감
  │        (토스 SDK)         확인 모달
  │              │               │
  │              ▼               ▼
  │        결제 성공/실패    차감 성공/실패
  │              │               │
  │              └───────┬───────┘
  │                      │ 성공
  │                      ▼
  │               등록 완료 화면
  │               "관리자 승인 대기 중"
  │                      │
  │                      ▼
  │                   D-03 내 앱
  │
  ├─[진행 중 테스트 카드 클릭]──→ D-04
  │
  ├─[최근 피드백 클릭]──→ D-04 (피드백 탭)
  │
  └─[요약 카드 클릭]──→ D-03 (해당 필터)
```

### 3-3. 내 앱 관리 흐름

```
D-03 내 앱 관리
  │
  ├─[상태 필터]──→ ALL / RECRUITING / IN_TESTING / COMPLETED / PRODUCTION
  │
  ├─[앱 카드 클릭]──→ D-04 앱 상세
  │
  └─[+ 새 앱 등록]──→ D-02
```

### 3-4. 앱 상세 / 테스트 관리 흐름

```
D-04 앱 상세
  │
  ├─[탭: 개요]
  │   ├─ 앱 정보, D-Day, 테스터 현황
  │   ├─ [수정] → D-02 (편집 모드, PENDING/RECRUITING만)
  │   └─ [삭제] → 확인 모달 → D-03 (PENDING/RECRUITING만)
  │
  ├─[탭: 지원자 관리] (RECRUITING 상태)
  │   ├─ 지원자 목록 (신뢰도순)
  │   ├─ [승인] → Application→APPROVED → (전원 시 IN_TESTING)
  │   ├─ [거절] → Application→REJECTED
  │   └─ [전체 승인] → 상위 N명 일괄
  │
  ├─[탭: 참여자 모니터링] (IN_TESTING 상태)
  │   ├─ 참여자 상태 (초록/주황/빨강)
  │   ├─ [메시지 발송] → 메시지 모달
  │   └─ 이탈 알림 카드
  │
  ├─[탭: 피드백] (COMPLETED 이후)
  │   ├─ 평균 별점 + 항목별
  │   ├─ 피드백 리스트
  │   └─ [필터: 전체/피드백/버그]
  │
  └─[탭: 프로덕션] (COMPLETED 상태)
      ├─ 체크리스트 (자동+수동)
      ├─ Google 답변 템플릿
      └─ [프로덕션 확인] → 최종 확인 모달 → 리워드 자동 지급
```

### 3-5. 결제/구독 흐름

```
D-05 결제/구독
  │
  ├─[탭: 플랜]
  │   ├─ 4개 플랜 비교표
  │   ├─ [업그레이드] → 결제 확인 모달 → PG
  │   ├─ [다운그레이드] → 확인 모달 ("다음 결제일부터")
  │   └─ [해지] → 확인 모달 → 해지 완료
  │
  ├─[탭: 결제 수단]
  │   ├─ 등록된 수단 목록
  │   ├─ [추가] → PG SDK → 빌링키 저장
  │   ├─ [기본 설정] → 기본 수단 변경
  │   └─ [삭제] → 확인 모달 (구독 중 기본 수단 삭제 불가)
  │
  └─[탭: 결제 내역]
      ├─ 날짜, 내용, 금액, 상태
      └─ [영수증] → PG 영수증 새 탭
```

## 4. 테스터 흐름

### 4-1. 메인 네비게이션

```
[상단 Nav]
  ├─ 로고 (→ T-01)
  ├─ 앱 탐색 (→ T-01)
  ├─ 내 테스트 (→ T-03)
  ├─ 리워드 (→ T-05)
  ├─ [역할 토글: 개발자 모드] (→ D-01)
  ├─ 🔔 알림 (→ 알림 센터)
  └─ 프로필 (→ S-04)
```

### 4-2. 앱 탐색 → 지원 흐름

```
T-01 테스터 홈 (앱 탐색)
  │
  ├─[검색 바]──→ 검색 결과 (같은 화면)
  │
  ├─[필터/정렬]
  │   ├─ 카테고리 칩
  │   ├─ 리워드 유형 (유료/상호)
  │   └─ 정렬 (최신/리워드높은순/인기)
  │
  ├─[HOT 태그 앱]──→ T-02
  │
  └─[앱 카드 클릭]──→ T-02 앱 상세
                        │
                        ├─ 스크린샷 갤러리 (스와이프)
                        ├─ 테스트 정보 (기간, 리워드, 남은인원)
                        ├─ 테스트 가이드
                        │
                        ├─[테스트 지원하기]
                        │     │
                        │  ┌──┴──────────────┐
                        │  │ 지원 가능        │ 지원 불가
                        │  ▼                  ▼
                        │ 기기/메시지 입력    에러 메시지
                        │ 팝업               (본인앱/중복/
                        │  │                  동시초과/마감)
                        │  ├─[지원]
                        │  │  │
                        │  │  ├─성공 → "지원완료" 토스트
                        │  │  │        버튼→"지원완료"
                        │  │  │
                        │  │  └─대기열 → "대기 등록" 토스트
                        │  │
                        │  └─[취소] → 팝업 닫기
                        │
                        └─[이미 지원] → 버튼 "지원 완료" (비활성)
```

### 4-3. 내 테스트 흐름

```
T-03 내 테스트
  │
  ├─[탭: 진행 중]
  │   ├─ 카드: 아이콘, 이름, D-Day, 오늘실행여부
  │   ├─ [Google Play 열기] → 새 탭 (실행기록 저장)
  │   └─ [피드백 작성] → T-04 (COMPLETED/기한 내)
  │
  ├─[탭: 완료]
  │   ├─ 카드: 이름, 리워드, 지급상태
  │   ├─ [피드백 보기] → T-04 (읽기 모드)
  │   └─ [피드백 수정] → T-04 (수정 모드, 기한 내)
  │
  └─[탭: 대기 중]
      ├─ 카드: 이름, 상태(PENDING/WAITLISTED)
      └─ [지원 취소] → 확인 모달 → PENDING에서만
```

### 4-4. 피드백 작성 흐름

```
T-04 피드백 작성
  │
  ├─① 전체 만족도 (1~5 별점) ───── 필수
  │
  ├─② 항목별 평가 ──────────────── 필수
  │   (개발자 설정 항목만 표시)
  │   UI/UX, 성능, 기능, 안정성
  │
  ├─③ 텍스트 피드백 ─────────────── 조건부 필수
  │   (feedbackRequired=true 시)
  │   최소 10자, 비속어 경고
  │
  ├─④ 버그 리포트 ──────────────── 선택
  │   제목 → 설명 필수
  │   스크린샷 첨부 (최대 3장)
  │   기기 정보 자동
  │
  └─[제출]
      │
      ├─성공 → 토스트 "피드백 제출 완료" → T-03
      └─실패 → 인라인 에러 (미입력 항목 하이라이트)
```

### 4-5. 리워드 흐름

```
T-05 리워드/포인트
  │
  ├─잔액 표시: {N}P ({N}원)
  │
  ├─[탭: 포인트 내역]
  │   ├─ 적립/사용 리스트 (무한 스크롤)
  │   └─ 색상: 초록(적립)/빨강(출금)/주황(교환)
  │
  ├─[탭: 출금]
  │   ├─ [출금 신청] → 출금 폼
  │   │    ├─ 방식: 계좌이체 / PayPal
  │   │    ├─ 계좌 정보 입력
  │   │    ├─ 금액 (최소 10,000)
  │   │    └─ [신청] → 확인 모달 → 완료 토스트
  │   │
  │   └─ 출금 내역 리스트
  │
  ├─[탭: 기프티콘]
  │   ├─ 상품 그리드
  │   └─ [교환] → 확인 모달 → "5분 내 발송"
  │
  └─[탭: 크레딧]
      ├─ 크레딧 잔액
      └─ 크레딧 내역 (+20C / +10C / -50C)
```

## 5. 공통 흐름

### 5-1. 마이페이지 흐름

```
S-04 마이페이지
  │
  ├─[섹션: 프로필]
  │   ├─ 닉네임, 이미지, 자기소개
  │   └─ [편집] → 프로필 편집 모달/인라인
  │
  ├─[섹션: 역할]
  │   ├─ 현재 역할 표시
  │   ├─ [개발자 모드] → D-01
  │   └─ [테스터 모드] → T-01
  │
  ├─[섹션: 활동 통계] (P2)
  │   ├─ 개발자: 앱 수/프로덕션/진행 중
  │   └─ 테스터: 완료/완료율/총 리워드
  │
  ├─[섹션: 구독/크레딧] (P2)
  │   ├─ 플랜, 남은 등록 수, 크레딧
  │   └─ [변경] → D-05
  │
  ├─[섹션: 알림 설정] (P2)
  │   ├─ 푸시 ON/OFF
  │   ├─ 이메일 ON/OFF
  │   └─ SMS ON/OFF
  │
  ├─[고객센터] → CS-01
  │
  ├─[로그아웃] → 확인 모달 → S-01
  │
  └─[회원탈퇴] (P2) → 탈퇴 확인 화면
      ├─ 진행 중 테스트 경고
      ├─ 미출금 포인트 경고
      ├─ "탈퇴합니다" 입력
      └─ [탈퇴] → S-01
```

### 5-2. 알림 센터 흐름

```
🔔 알림 아이콘 (Nav)
  │
  ├─ 미읽은 수 뱃지
  │
```
