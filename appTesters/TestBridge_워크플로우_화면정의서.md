# TestBridge 워크플로우 & 화면 정의서

---

## 1. 사용자 유형

| 유형 | 설명 |
|------|------|
| 개발자 (Developer) | 앱을 등록하고 테스터를 모집하는 사용자 |
| 테스터 (Tester) | 앱을 테스트하고 리워드를 받는 사용자 |
| 개발자+테스터 (Dual) | 상호 테스트를 위해 양쪽 역할을 모두 수행하는 사용자 |

---

## 2. 전체 서비스 플로우 (End-to-End)

```

   │                                    │                                  │
   ├─ 1. 회원가입/로그인 ──────────────►│◄────────── 1. 회원가입/로그인 ──┤
   │                                    │                                  │
   ├─ 2. 앱 등록 ─────────────────────►│                                  │
   │   (앱 소개, 스크린샷, 카테고리)     │                                  │
   │                                    │                                  │
   ├─ 3. 테스트 모집 공고 생성 ────────►│──── 4. 모집 중인 앱 목록 노출 ──►│
   │   (인원, 기간, 리워드 설정)         │                                  │
   │                                    │                                  │
   │                                    │◄───── 5. 테스트 지원 ────────────┤
   │                                    │                                  │
   ├─ 6. 지원자 확인 & 선정 ──────────►│──── 7. 선정 알림 발송 ──────────►│
   │                                    │                                  │
   ├─ 8. Google Play 테스트 링크 공유 ─►│──── 9. 링크 전달 ──────────────►│
   │                                    │                                  │
   │                                    │◄───── 10. 앱 설치 & 옵트인 ─────┤
   │                                    │                                  │
   │   11. 테스트 현황 모니터링 ◄───────│──── 12. 14일간 테스트 참여 ─────►│
   │   (대시보드)                        │   (일일 실행, 피드백 작성)        │
   │                                    │                                  │
   │                                    │◄───── 13. 피드백/리뷰 제출 ─────┤
   │                                    │                                  │
   ├─ 14. 피드백 확인 ◄────────────────│                                  │
   │                                    │                                  │
   ├─ 15. 프로덕션 등록 신청 ──────────►│                                  │
   │   (Google Play Console)            │                                  │
   │                                    │                                  │
   ├─ 16. 등록 성공 확인 ─────────────►│──── 17. 리워드 지급 ────────────►│
   │                                    │                                  │
   └────────────────────────────────────┘──────────────────────────────────┘
```

## 3. 화면 정의

### 3.1 공통 화면

#### S-01. 온보딩 / 랜딩 페이지
- **목적**: 서비스 소개 및 회원가입 유도
- **구성요소**:
  - 서비스 핵심 가치 소개 (히어로 섹션)
  - "개발자로 시작" / "테스터로 시작" CTA 버튼
  - 서비스 이용 플로우 시각화 (3~4단계)
  - 성공 사례 / 리뷰 섹션
  - FAQ 섹션

#### S-02. 회원가입
- **목적**: 계정 생성
- **구성요소**:
  - 소셜 로그인 (Google 필수, 카카오, 네이버 선택)
  - 역할 선택: 개발자 / 테스터 / 둘 다
  - 기본 프로필 입력 (닉네임, 프로필 이미지)
  - 이용약관 동의
- **참고**: Google 계정 로그인 필수 (Play Console 연동을 위해)

#### S-03. 로그인
- **목적**: 기존 사용자 로그인
- **구성요소**:
  - 소셜 로그인 버튼들
  - 비밀번호 찾기 (이메일 로그인 시)

#### S-04. 마이페이지 / 프로필
- **목적**: 내 정보 관리
- **구성요소**:
  - 프로필 편집 (닉네임, 소개, 프로필 이미지)
  - 역할 전환 (개발자 ↔ 테스터)
  - 구독 플랜 정보 / 변경
  - 크레딧 잔액 (상호 테스트 크레딧)
  - 리워드 포인트 잔액 (테스터)
  - 알림 설정
  - 로그아웃 / 회원탈퇴

#### S-05. 알림 센터
- **목적**: 모든 알림 모아보기
- **구성요소**:
  - 알림 리스트 (선정, 테스트 시작/종료, 리워드 지급, 이탈 경고 등)
  - 읽음/안읽음 구분
  - 알림 클릭 시 해당 화면으로 이동

### 3.2 개발자 화면

#### D-01. 개발자 대시보드 (홈)
- **목적**: 내 앱과 테스트 현황 한눈에 파악
- **구성요소**:
  - 진행 중인 테스트 요약 카드 (앱명, D-Day, 참여자 수/목표)
  - 최근 피드백 미리보기
  - 구독 플랜 & 남은 등록 가능 앱 수
  - 크레딧 잔액
  - 빠른 액션: "새 앱 등록" 버튼

#### D-02. 앱 등록
- **목적**: 테스트할 앱 정보 입력
- **구성요소**:
  - Step 1 - 기본 정보
    - 앱 이름
    - 패키지명 (com.example.app)
    - 카테고리 (게임, 유틸리티, 소셜 등)
    - 앱 설명 (간단 소개)
    - 앱 아이콘 업로드
    - 스크린샷 업로드 (최대 5장)
  - Step 2 - 테스트 설정
    - 테스트 유형: 유료 리워드 / 상호 테스트(크레딧)
    - 필요 테스터 수 (최소 20명 권장)
    - 테스트 기간 (14일 고정 + 추가 가능)
    - Google Play 비공개 테스트 링크
  - Step 3 - 리워드 설정 (유료 리워드 선택 시)
    - 리워드 유형: 기본 / 피드백 포함 / 심화
    - 1인당 리워드 금액 설정
    - 총 예상 비용 계산 표시
    - 결제 진행
  - Step 4 - 피드백 설정
    - 피드백 필수 여부
    - 피드백 항목 커스텀 (별점, 텍스트, 버그 리포트 등)
    - 테스터에게 전달할 테스트 가이드 작성
  - "등록하기" 버튼 → 검토 후 공개

#### D-03. 내 앱 목록
- **목적**: 등록한 앱 전체 관리
- **구성요소**:
  - 앱 카드 리스트 (앱 아이콘, 이름, 상태, 진행률)
  - 상태 필터: 모집 중 / 테스트 진행 중 / 테스트 완료 / 프로덕션 등록 완료
  - 각 카드 클릭 시 → D-04 상세 화면으로 이동

#### D-04. 앱 상세 / 테스트 관리
- **목적**: 특정 앱의 테스트 진행 상황 상세 관리
- **구성요소**:
  - 탭 1: 현황 (Overview)
    - 테스트 상태 (모집 중 / 진행 중 / 완료)
    - D-Day 카운트다운 (14일 중 n일차)
    - 참여자 현황: 옵트인 유지 n명 / 목표 n명 (프로그레스 바)
    - 이탈자 알림 (빨간 경고)
  - 탭 2: 지원자 관리
    - 지원자 리스트 (프로필, 신뢰도 점수, 테스트 이력)
    - 승인 / 거절 버튼
    - 대기열 (여유 인원)
  - 탭 3: 참여자 모니터링
    - 참여자별 상태 (옵트인 유지 / 이탈)
    - 참여자별 최근 앱 실행 일시
    - 참여자에게 메시지 발송
  - 탭 4: 피드백
    - 수신된 피드백 리스트
    - 별점 평균
    - 버그 리포트 목록
    - 피드백별 상세 보기
  - 탭 5: 프로덕션 가이드
    - 프로덕션 등록 체크리스트
    - Google 질문 답변 가이드 (템플릿 제공)
    - "프로덕션 등록 완료" 확인 버튼 → 리워드 지급 트리거

#### D-05. 결제 / 구독 관리
- **목적**: 구독 플랜 선택 및 결제
- **구성요소**:
  - 플랜 비교표 (Free / Basic / Pro / Enterprise)
  - 현재 플랜 표시
  - 업그레이드 / 다운그레이드 버튼
  - 결제 수단 관리 (카드, 토스페이, 카카오페이)
  - 결제 내역 / 영수증 조회
  - 건별 결제 내역

#### D-06. 메시지 (개발자 측)
- **목적**: 테스터와 1:1 소통
- **구성요소**:
  - 대화 목록 (앱별 / 테스터별)
  - 채팅 화면
  - 전체 공지 발송 (선정된 테스터 전원에게)

### 3.3 테스터 화면

#### T-01. 테스터 홈 / 앱 탐색
- **목적**: 테스트 가능한 앱 찾기
- **구성요소**:
  - 검색바
  - 필터: 카테고리, 리워드 금액, 리워드 유형(현금/크레딧), 인기순, 최신순
  - 앱 카드 리스트
    - 앱 아이콘
    - 앱 이름
    - 카테고리 태그
    - 리워드 금액 표시
    - 남은 모집 인원
    - 개발자 정보
  - 카드 클릭 → T-02로 이동

#### T-02. 앱 상세 (테스터 뷰)
- **목적**: 앱 정보 확인 및 테스트 지원
- **구성요소**:
  - 앱 아이콘, 이름, 카테고리
  - 앱 설명 (개발자 작성)
  - 스크린샷 갤러리 (스와이프)
  - 테스트 정보
    - 테스트 기간: 14일
    - 리워드 금액 / 유형
    - 필요 참여 조건 (피드백 필수 여부 등)
    - 남은 모집 인원 / 전체 인원
  - 개발자 프로필 (닉네임, 등록 앱 수)
  - "테스트 지원하기" 버튼
  - 지원 시 간단 자기소개 / 보유 기기 정보 입력 팝업

#### T-03. 내 테스트 현황
- **목적**: 참여 중인 테스트 관리
- **구성요소**:
  - 탭: 진행 중 / 완료 / 지원 중 (대기)
  - 진행 중 테스트 카드
    - 앱 아이콘, 이름
    - D-Day (n일차 / 14일)
    - 프로그레스 바
    - 오늘 앱 실행 여부 (체크 표시)
    - "피드백 작성" 버튼
    - "Google Play에서 열기" 버튼
  - 완료된 테스트 카드
    - 리워드 수령 상태 (대기 중 / 지급 완료)
    - 내가 작성한 피드백 보기
  - 지원 중 (대기)
    - 지원한 앱 목록
    - 선정 대기 / 거절 상태 표시

#### T-04. 피드백 작성
- **목적**: 테스트 중인 앱에 대한 피드백 제출
- **구성요소**:
  - 전체 만족도 별점 (1~5)
  - 항목별 평가 (개발자가 설정한 항목)
    - UI/UX
    - 성능/속도
    - 기능 완성도
    - 안정성 (크래시 여부)
  - 텍스트 피드백 (자유 작성)
  - 버그 리포트 (선택)
    - 버그 제목
    - 재현 방법
    - 스크린샷 첨부
    - 기기 정보 (자동 수집 가능)
  - "제출하기" 버튼

#### T-05. 리워드 / 포인트 관리
- **목적**: 적립된 리워드 확인 및 출금
- **구성요소**:
  - 현재 포인트 잔액 (원화 환산 표시)
  - 적립/사용 내역 리스트
    - 날짜, 앱 이름, 금액, 유형 (적립/출금/교환)
  - 출금 신청
    - 출금 방식 선택 (계좌이체, PayPal)
    - 출금 금액 입력 (최소 10,000원)
    - 은행/계좌번호 입력
  - 기프티콘 교환
    - 교환 가능 기프티콘 목록
    - 교환 버튼

#### T-06. 신뢰도 / 프로필
- **목적**: 테스터로서의 활동 이력 및 신뢰도 확인
- **구성요소**:
  - 신뢰도 점수 (0~100)
  - 현재 등급 배지 (브론즈 / 실버 / 골드 / 다이아몬드)
  - 다음 등급까지 남은 포인트
  - 활동 통계
    - 총 테스트 참여 수
    - 완료율 (%)
    - 평균 피드백 점수
    - 제출한 버그 리포트 수
  - 테스트 이력 리스트

#### T-07. 메시지 (테스터 측)
- **목적**: 개발자와 1:1 소통
- **구성요소**:
  - 대화 목록
  - 채팅 화면

### 3.4 관리자 화면 (Admin)

#### A-01. 관리자 대시보드
- **목적**: 플랫폼 전체 현황 모니터링
- **구성요소**:
  - 핵심 지표: 총 사용자 수, DAU/MAU, 진행 중 테스트 수, 총 거래액
  - 신규 가입 추이 차트
  - 앱 등록 추이 차트
  - 수익 현황 (구독료 + 수수료)
  - 신고 접수 건수

#### A-02. 사용자 관리
- **목적**: 사용자 조회, 제재, 관리
- **구성요소**:
  - 사용자 검색 (이름, 이메일, 역할)
  - 사용자 상세 (프로필, 활동 이력, 결제 내역)
  - 제재 (경고, 일시정지, 영구정지)
  - 신고 접수 내역 처리

#### A-03. 앱 관리
- **목적**: 등록된 앱 검토 및 관리
- **구성요소**:
  - 앱 등록 승인/반려
  - 부적절한 앱 신고 처리
  - 앱 상세 조회

#### A-04. 정산 관리
- **목적**: 리워드 정산 및 수수료 관리
- **구성요소**:
  - 리워드 지급 대기 목록
  - 출금 신청 처리
  - 수수료 수익 현황
  - 구독료 수익 현황

## 4. 화면 흐름도 (Screen Flow)

### 4.1 개발자 주요 플로우

```
온보딩(S-01) → 회원가입(S-02) → 개발자 대시보드(D-01)
                                       │
                          ┌─────────────┼─────────────┐
                          ▼             ▼             ▼
                    앱 등록(D-02)  내 앱 목록(D-03)  구독관리(D-05)
                          │             │
                          ▼             ▼
                    모집 시작      앱 상세(D-04)
                                       │
                          ┌─────┬───────┼───────┬─────┐
                          ▼     ▼       ▼       ▼     ▼
                        현황  지원자  참여자  피드백  프로덕션
                              관리    모니터링        가이드
                                                       │
                                                       ▼
                                                  리워드 지급
```

### 4.2 테스터 주요 플로우

```
온보딩(S-01) → 회원가입(S-02) → 테스터 홈(T-01)
                                       │
                          ┌─────────────┼─────────────┐
                          ▼             ▼             ▼
                    앱 탐색(T-01)  내 테스트(T-03)  리워드(T-05)
                          │             │             │
                          ▼             ▼             ▼
                    앱 상세(T-02)  피드백 작성(T-04)  출금/교환
                          │

                          │
                          ▼

                    앱 설치 & 14일 테스트
```

## 6. MVP 핵심 워크플로우

MVP에서 반드시 동작해야 하는 핵심 플로우 3가지:

### 플로우 1: 개발자의 앱 등록 ~ 테스터 선정
```
S-02 회원가입 → D-01 대시보드 → D-02 앱 등록(4 Steps) → D-03 내 앱 목록
→ D-04 앱 상세 > 지원자 관리 탭 → 테스터 승인
```

### 플로우 2: 테스터의 앱 탐색 ~ 테스트 완료
```
S-02 회원가입 → T-01 앱 탐색 → T-02 앱 상세 → 테스트 지원
→ S-05 선정 알림 → T-03 내 테스트 현황 → T-04 피드백 작성
→ T-03 테스트 완료 확인
```

### 플로우 3: 테스트 완료 ~ 리워드 지급
```
D-04 앱 상세 > 현황 탭 (14일 완료 확인)
→ D-04 프로덕션 가이드 탭 → "프로덕션 등록 완료" 확인
→ 리워드 자동/수동 지급 → T-03 테스터 리워드 수령 확인
```
