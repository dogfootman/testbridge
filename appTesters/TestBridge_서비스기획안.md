# TestBridge - 안드로이드 앱 테스터 매칭 플랫폼 기획안

## 1. 서비스 개요

**서비스명(가칭):** TestBridge  
**한줄 소개:** 안드로이드 개인 개발자와 테스터를 연결하여 Google Play 비공개 테스트 요건(14일 / 14명)을 충족시켜주는 매칭 플랫폼  
**플랫폼:** 웹사이트 + 모바일 앱 (반응형 웹 우선 → 추후 앱 확장)

---

## 2. 핵심 문제 정의

Google Play는 2023년 11월부터 신규 개인 개발자 계정에 대해 프로덕션 배포 전 **비공개 테스트 트랙에서 20명 중 14명 이상이 14일 연속 옵트인**을 유지하도록 요구하고 있습니다.

**개발자의 고충:**
- 주변에 20명의 테스터를 구하기 어려움
- 14일간 지속적으로 참여시키기 더 어려움
- 커뮤니티에 테스터 모집 글을 올려도 반응이 저조
- 테스터 이탈 시 다시 처음부터 카운트

**테스터의 니즈:**
- 새로운 앱을 남들보다 먼저 체험하고 싶은 욕구
- 부수입 또는 리워드를 받을 수 있는 기회
- 앱 개발 생태계에 기여하고 싶은 마음

---

## 3. 서비스 플로우

### 3.1 개발자 플로우

```
회원가입/로그인
    ↓
앱 등록 (앱 소개, 스크린샷, 카테고리, 테스트 기간 등)
    ↓
테스터 모집 공고 생성 (필요 인원, 리워드 조건 설정)
    ↓
지원자 확인 및 테스터 선정 (20명 이상)
    ↓
Google Play 비공개 테스트 링크 공유
    ↓
14일간 테스트 진행 (참여 현황 대시보드로 모니터링)
    ↓
프로덕션 등록 성공 확인
    ↓
리워드 지급 실행
```

### 3.2 테스터 플로우

```
회원가입/로그인
    ↓
테스트 가능한 앱 목록 탐색 (카테고리, 리워드별 필터)
    ↓
관심 있는 앱에 테스트 지원
    ↓
선정 알림 수신
    ↓
앱 설치 및 14일간 테스트 참여
    ↓
(선택) 피드백/리뷰 작성
    ↓
테스트 완료 후 리워드 수령
```

## 4. 주요 기능

### 4.1 개발자용 기능

| 기능 | 설명 |
|------|------|
| 앱 등록 | 앱 이름, 설명, 스크린샷, APK/AAB 다운로드 링크, 카테고리 등록 |
| 테스터 모집 공고 | 필요 인원 수, 테스트 기간, 리워드 조건 설정 |
| 지원자 관리 | 지원자 프로필 확인, 승인/거절, 메시지 전송 |
| 테스트 대시보드 | 참여자별 옵트인 상태, D-Day 카운트다운, 이탈 알림 |
| 리워드 관리 | 테스트 완료 확인 후 리워드 일괄/개별 지급 |
| 피드백 수집 | 테스터가 남긴 버그 리포트, 개선 의견 열람 |

### 4.2 테스터용 기능

| 기능 | 설명 |
|------|------|
| 앱 탐색 | 카테고리, 리워드, 인기순 등 필터링 |
| 테스트 지원 | 원클릭 지원 + 간단한 자기소개 |
| 테스트 현황 | 내가 참여 중인 테스트 목록, 남은 일수, 리워드 현황 |
| 피드백 작성 | 별점, 텍스트 리뷰, 버그 리포트 양식 |
| 리워드 내역 | 적립/사용 내역, 출금 |
| 신뢰도 점수 | 테스트 완료율 기반 신뢰도 배지 (성실한 테스터일수록 선정 우선) |

### 4.3 공통 기능

| 기능 | 설명 |
|------|------|
| 알림 시스템 | 선정, 테스트 시작/종료, 리워드 지급 등 푸시/이메일 알림 |
| 채팅/메시지 | 개발자-테스터 간 1:1 소통 |
| 신고/차단 | 악성 사용자 신고 및 차단 |

## 5. 리워드 시스템 (제안)

리워드는 플랫폼의 핵심 동력이므로 다양한 옵션을 병행하는 것을 권장합니다.

### 5.1 추천 리워드 구조

**Option A: 포인트 기반 (권장)**
- 테스트 완료 시 포인트 지급 (개발자가 금액 설정)
- 포인트 → 현금 전환 (최소 출금 금액 설정)
- 포인트 → 기프티콘/쿠폰 교환
- 장점: 플랫폼 내 경제 구축 가능, 수수료 모델과 연동 용이

**Option B: 상호 테스트 (Peer Testing)**
- 개발자가 다른 개발자의 앱을 테스트해주면, 자신의 앱 테스터 크레딧 획득
- 무료로 운영 가능, 개발자 커뮤니티 형성에 유리
- 단점: 순수 테스터(비개발자) 유입이 어려움

**Option C: 하이브리드 (A + B 결합, 최종 권장)**
- 개발자는 포인트 리워드 또는 상호 테스트 중 선택 가능
- 테스터는 포인트 적립 또는 자신의 앱 테스트 크레딧 획득
- 가장 넓은 사용자층 확보 가능

### 5.2 리워드 금액 가이드라인

| 테스트 유형 | 권장 리워드 (1인당) |
|------------|-------------------|
| 기본 (14일 옵트인만) | 1,000 ~ 3,000원 |
| 피드백 포함 | 3,000 ~ 5,000원 |
| 심화 (버그 리포트 + 상세 리뷰) | 5,000 ~ 10,000원 |

## 6. 수익 모델 (제안)

여러 수익원을 단계적으로 도입하는 전략을 권장합니다.

### Phase 1: 초기 (사용자 확보 단계)
- **무료 기본 플랜**: 월 1개 앱 등록, 기본 대시보드
- **수수료 모델**: 리워드 지급 시 10~15% 플랫폼 수수료
  - 예: 개발자가 테스터에게 3,000원 리워드 설정 → 실제 결제 3,300~3,450원

### Phase 2: 성장기
- **프리미엄 구독**: 월 9,900원 ~ 19,900원
  - 무제한 앱 등록
  - 테스터 우선 매칭 (신뢰도 높은 테스터 우선 배정)
  - 상세 분석 리포트
  - 수수료 할인 (10% → 5%)
- **긴급 모집 부스트**: 1회 3,000~5,000원
  - 테스터 모집 공고를 상단에 노출

### Phase 3: 확장기
- **광고**: 앱 탐색 페이지 내 관련 개발 도구/서비스 광고
- **B2B 서비스**: 기업 개발팀 대상 대량 테스트 관리 솔루션

## 7. 핵심 차별화 포인트

### 7.1 신뢰도 시스템
테스터의 이탈은 개발자에게 치명적입니다. 신뢰도 시스템으로 이를 방지합니다.

- **테스터 신뢰도 점수**: 테스트 완료율, 피드백 품질, 참여 기간 등 기반
- **배지 시스템**: 브론즈 → 실버 → 골드 → 다이아몬드
- **혜택 차등**: 높은 등급 테스터는 더 높은 리워드의 테스트에 우선 지원 가능
- **페널티**: 중도 이탈 시 신뢰도 하락, 일정 기간 지원 제한

### 7.2 테스트 보험
- 테스터 이탈 시 자동으로 대기 테스터에게 알림 발송
- 개발자는 여유 인원(예: 25명) 선정 가능 (최소 20명 충족 보장)

### 7.3 자동화
- Google Play Console 연동 (가능한 범위 내)으로 옵트인 상태 자동 추적
- 14일 완료 시 자동 리워드 지급 옵션

## 9. MVP (최소 기능 제품) 범위

첫 출시에 포함할 핵심 기능만 정리합니다.

### MVP에 포함
- 개발자/테스터 회원가입 (Google 소셜 로그인)
- 앱 등록 및 테스트 모집 공고
- 테스터 지원 및 선정
- 기본 테스트 현황 대시보드 (D-Day, 참여자 수)
- 알림 (이메일)
- 간단한 피드백 양식
- 포인트 리워드 (수동 지급)

### MVP에서 제외 (추후 개발)
- 상호 테스트(Peer Testing) 시스템
- 자동화 (Google Play Console 연동)
- 채팅 기능
- 프리미엄 구독
- 모바일 앱
- 신뢰도 배지 시스템
- 결제/출금 시스템 (초기에는 계좌이체 등 수동 처리)
